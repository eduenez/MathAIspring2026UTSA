# GEMINI.md - MAT 4953/MAT 6973 Mathematical Foundations of AI

This file provides foundational mandates and architectural guidance for Gemini CLI when working in this repository.

## Project Context
- **Course Website**: A Jekyll-based static site for the "MAT 4953/MAT 6973 Mathematical Foundations of AI" course at UTSA (Spring 2026).
- **Educational Code**: Supplementary Python scripts and Jupyter notebooks (MNIST, SVD, bias-variance) located in the `code/` directory.
- **Tech Stack**: Jekyll (Ruby), Python 3, Jupyter, MathJax (for LaTeX rendering).

## Foundational Mandates
- **No `_site/` Edits**: Never modify files in the `_site/` directory; it is auto-generated by Jekyll.
- **Content Structure**: All course content (modules, assignments, announcements) resides in `collections/` using Markdown with YAML front matter.
- **Math Rendering**: Ensure `math: mathjax3` is present in the YAML front matter for any page requiring LaTeX math rendering.
- **Surgical Content Updates**: When adding or modifying course materials, use the existing collection patterns (e.g., `collections/_modules/week-XX.md`).
- **Style Management**: Custom styles must be added to `_sass/custom/` and imported in `_sass/custom/custom.scss`.
- **Navigation**: Main navigation and footer links are managed in `_includes/nav_footer_custom.html`.

## Technical Standards & Workflow

### Jekyll / Ruby
- **Local Preview**: Use the following command to test site changes locally:
  ```bash
  export PATH=$PATH:$(ruby -e 'print Gem.user_dir')/bin
  bundle exec jekyll serve
  ```
  The site will be available at [http://localhost:4000/MathAIspring2026UTSA/](http://localhost:4000/MathAIspring2026UTSA/).
- **Dependencies**: Managed via `Gemfile`. Run `bundle install` after updates. Gems are installed locally in `vendor/bundle`.

### Python / AI Code
- **Directory**: All code and notebooks belong in `code/`.
- **Dependencies**: Managed via `requirements.txt`.
- **Notebooks**: When updating `.ipynb` files, ensure they are in a clean state (cleared outputs or fully executed as appropriate for the task).

### Git & Deployment
- **Exclusions**: Do not commit `_site/`, `.venv/`, or backup files (`*~`, `#*`).
- **GitHub Pages**: The site is deployed via GitHub Pages from the `main` branch.

## Research & Validation
- **Verification**: After modifying Jekyll files, verify the build by running `bundle exec jekyll build` or `serve`.
- **Links**: Ensure all relative links between modules, assignments, and assets are valid.
- **Code Validation**: If modifying Python scripts, run them to ensure no regressions. For notebooks, verify they open and run in a standard Jupyter environment.
